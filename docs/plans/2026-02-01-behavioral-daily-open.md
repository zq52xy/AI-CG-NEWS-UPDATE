# 让用户更愿意每天打开 AI-CG-NEWS — 行为设计方案

**框架**：behavioral-product-design  
**目标行为**：用户更愿意「每天打开」AI-CG-NEWS  
**约束**：静态站、无后端、无 Push；仅 localStorage + 前端可改。

---

## 一、目标行为与习惯回路

### 1.1 目标行为

- **希望用户做的**：每天（至少经常）打开一次 AI-CG-NEWS，看当日/近期资讯。
- **成功标准**：打开频次提升、连续打开天数可感知（不一定要强求「每日」）。

### 1.2 习惯回路（Cue → Routine → Reward）

| 环节 | 现状 | 可强化点 |
|------|------|----------|
| **Cue 触发** | 无 Push/邮件；仅靠用户自己记得或书签 | 固定时间认知（「每天 08:00 更新」）、轻量「回访提示」、PWA/主屏 |
| **Routine 惯例** | 打开 → 看列表 → 点链接/收藏 | 默认就是「今天」、减少一次点击到「今日新内容」 |
| **Reward 奖励** | 新内容、收藏列表、分屏预览 | 让「今日更新」可见、轻量连续打开反馈（非强迫） |

---

## 二、行为障碍诊断

### 2.1 用户为什么「不每天打开」？

- **记不住**：没有提醒，容易忘。
- **不知道有没有新的**：不知道今天是否已更新，打开动机弱。
- **打开后没有「今天专属」感**：若默认或首屏不强调「今天」，容易觉得「和昨天一样」。
- **没有「失去感」**：不打开也不会觉得丢了什么（收藏在、历史在，但无「连续打开」等轻量投入）。

### 2.2 当前产品里已有的助力

- 历史里有「今天」标记（📌 今天），且默认会选最新日期（通常即今天）→ **现状偏见**已部分在用。
- 收藏在侧边栏 → 有「我的东西在这里」的**禀赋效应**。
- 每日 08:00 更新 → 可强化为**固定时间 cue**（文案 + 小提示）。

---

## 三、行为原则与干预方向

### 3.1 损失厌恶（Loss Aversion）

- **原则**：「一旦形成 7 天左右的投入，损失厌恶会起作用，留存会上去。」
- **应用**：不做惩罚型设计，只做「可见的轻量投入」：
  - **连续打开天数**（仅展示，不断开不惩罚）：如「已连续打开 3 天」。
  - 数据存 localStorage（上次访问日期 + 连续天数），无账号、无服务器。
- **注意**：不搞「断签就清空」的强惩罚，避免 dark pattern；强调「你正在养成习惯」而非「你失败了」。

### 3.2 现时偏好 / 让「今天」更显眼（Present Bias）

- **原则**：人们更关注「现在」「今天」；今天有新东西 = 更强的打开动机。
- **应用**：
  - **默认始终是「今天」**：无 hash 时默认选最新日期（已是）；有 hash 时若为过去某天，顶栏或侧边栏弱提示「返回今日」。
  - **「今日」视觉强化**：顶栏或标题旁轻量文案/角标，如「今日已更新 · 共 N 条」或「今日」小标签。
  - **历史列表里「今天」置顶且高亮**：已有「今天」标记，可再加「新」或「最新」小角标。

### 3.3 减少摩擦（Friction）

- **原则**：每次多一次选择、多一次点击，都会降低「每天打开」的概率。
- **应用**：
  - **一次打开就看到今日内容**：保持「默认最新 = 今天」；避免首次打开先选日期。
  - **收藏/偏好本地存**：已有，保持不变。
  - **可选**：PWA 或「添加到主屏」引导（一次设置，之后从主屏/桌面打开），降低「找书签」的摩擦。

### 3.4 聪明默认（Defaults）

- **原则**：用户多数会沿用默认。
- **应用**：
  - 默认日期 = 最新（今天）；默认不自动展开预览、不自动弹窗，保持当前即可。
  - **首次访问**：可做一次性的轻量提示，如「每日 08:00 更新，建议收藏或设为首页」，不打断、不强制。

### 3.5 暂停时刻 / 小庆祝（Pause Moments）

- **原则**：用轻量动效或文案制造「你做了对的事」的瞬间，强化正向感受。
- **应用**：
  - **新一天首次打开**：若检测到「上次访问是昨天（或更早）、今天是新一天」，可轻量 toast/横幅一句，如「今日已更新，开始浏览吧」或「欢迎回来，今日共 N 条资讯」，不打断阅读。
  - **连续打开 3/7 天**：仅一次轻量文案，如「你已连续打开 7 天」，不弹窗、不打断，可放在顶栏旁或侧边栏底部小字。

## 四、可落地的功能与文案建议

### 4.1 低成本、高优先级（仅前端 + localStorage）

| 项目 | 做法 | 行为原理 |
|------|------|----------|
| **「今日」可见** | 顶栏或标题旁：当当前选中的是今天时，显示「今日 · 已更新」或「今日 N 条」 | 现时偏好、减少不确定性 |
| **连续打开天数** | localStorage 存 `lastVisitDate` + `streakDays`；每次打开时若为新的一天则 streak+1，否则若隔天则归 1；侧边栏底部或顶栏旁小字「已连续打开 N 天」 | 损失厌恶（轻量）、习惯可见化 |
| **固定时间 cue 文案** | 在空状态或页脚固定一句：「每日北京时间 08:00 更新」；可选：首次访问 toast「建议每天早晨打开查看」 | Cue 强化、设定预期 |
| **「返回今日」入口** | 当用户在看历史某天时，在顶栏或历史区域提供「今日」快捷入口，一键切回最新 | 减少摩擦、强化「今天」 |

### 4.2 中成本、可选

| 项目 | 做法 | 行为原理 |
|------|------|----------|
| **新一天首次打开的轻量庆祝** | 若 `lastVisitDate` < 今天，且用户刚打开，显示一句非模态文案（如顶部小横幅 3s 消失）：「今日已更新，共 N 条」 | 暂停时刻、奖励 |
| **连续 7 天轻量反馈** | 当 streak 达到 7 时，一次小文案：「已连续打开 7 天」；可不定期在侧边栏底部展示 streak | 损失厌恶、正向强化 |
| **PWA / 添加到主屏** | 提供「添加到主屏」或 PWA 安装引导（一次），降低后续打开成本 | 减少摩擦、cue 触达 |

### 4.3 文案与克制（避免 Dark Pattern）

- **不做**：断签清零、倒计时压迫、虚假「即将失去」、强制分享才续签。
- **要做**：文案中性、正向，如「已连续打开 N 天」「今日已更新」； streak 仅展示，断掉也不惩罚、不弹窗责备。
- **文化/情境**：当前以中文、北京时间为主；若以后做英文版，可保留同一逻辑、调整文案语气。

## 五、实现顺序建议

1. **Phase 1（优先）**
   - 顶栏/标题旁「今日」状态与「今日 N 条」。
   - localStorage 的 `lastVisitDate` + `streakDays`，侧边栏底部或顶栏旁「已连续打开 N 天」。
   - 固定一句「每日 08:00 更新」在页脚或空状态。
2. **Phase 2**
   - 「返回今日」快捷入口（当当前为历史日期时）。
   - 新一天首次打开的轻量欢迎/更新提示（非模态）。
3. **Phase 3（可选）**
   - 连续 7 天的轻量反馈。
   - PWA / 添加到主屏引导。

## 六、与产品评审的衔接

- 本方案与 `2026-02-01-product-review.md` 中的**产品定位、目标用户、功能清单**一致。

## 七、小结

| 行为原则 | 用法 |
|----------|------|
| **损失厌恶** | 轻量「连续打开天数」展示，不断签惩罚 |
| **现时偏好** | 强化「今日」、默认今天、「返回今日」入口 |
| **减少摩擦** | 默认今日、可选 PWA/主屏 |
| **聪明默认** | 默认最新日期、首次轻量「建议收藏」 |
| **暂停时刻** | 新一天轻量欢迎、连续 7 天一句肯定 |

目标：在**无 Push、无后端**的前提下，用**前端 + localStorage + 文案与轻量反馈**，让「每天打开」的 cue 更清晰、奖励更可见、轻量投入感（连续打开）更可感知，从而提高每日打开意愿。
